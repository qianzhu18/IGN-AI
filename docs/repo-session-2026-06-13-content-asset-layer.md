# 2026-06-13 官网内容资产层记录

## 背景

本轮讨论明确了一点：IGNAI 社区并不是独立于官网之外的新项目，而是当前官网缺少一层可沉淀、可复用、未来可上架的内容资产。

官网已有成员、活动、记录、加入、关于等功能入口，但对外介绍、合作方案、PPT 母稿、活动素材、成员故事、社区证明链路还不够完整。因此本轮先做内容层梳理，不直接改公开页面。

## 本轮完成

1. 新增 `docs/content/ignai-official-content-assets.zh-CN.md`
   - 写入 IGNAI 官方组织介绍的一句话、短版、中版、长版。
   - 写入对外合作方案文字版，包括合作定位、合作对象、合作形式、合作价值、原则和流程。
   - 梳理官方介绍 PPT、合作方案 PPT、融资 / 赞助方案 PPT、赛事 PPT、自我介绍 PPT 的母稿结构。
   - 梳理如果 IGNAI 真的作为社区运营，官网当前缺少的内容：官方叙事、社区证明、合作材料、活动运营内容、成员运营内容、内容生产机制、商业与组织基础。
   - 明确哪些内容可以近期上架，哪些应先内部整理，哪些需要素材后再上架。

2. 更新 `doc/roadmap/master-todo.md`
   - 把官网内容资产层首版梳理标为已完成。
   - 新增官网内容层补齐与内容资产库持续维护任务。

3. 更新 `doc/roadmap/schedule.md`
   - 加入官网内容资产层首版完成记录。
   - 加入后续文字 / 图片素材归档与公开页面筛选任务。
   - 加入对外材料 PPT 第一版生成任务。

## 社区面对外价值

- IGNAI 现在不只是“有官网功能”，而开始有一套可持续填充的官方内容资产层。
- 后续对外介绍、合作沟通、活动发布、成员展示、赞助沟通和赛事组织可以共用同一套母稿。
- 用户后续提供的文字和图片可以先进入内容资产层，再决定是否进入 About、Events、Records、Members、Join 或未来合作页。

## Upstreamable Pieces

本轮主要是 IGNAI 本地内容，不适合直接作为 NotionNext 上游 PR。

可间接沉淀为上游方向的是：

- 社区 / 组织站需要的不只是 `Post`，还需要内容资产治理、成员、活动、记录、合作材料等类型化内容。
- 未来可以在 NotionNext 文档中抽象为“community site content playbook”或“organization site content model”。

## Remaining Work

- 等待用户继续上传社区文字、图片、活动素材和成员故事。
- 基于内容资产层生成第一版官方介绍 PPT 和合作方案 PPT。
- 决定是否新增公开合作页，或先把合作入口放在 About / Join 页面。
- 把可公开的活动照片、成员项目、开源 PR、公众号文章整理成官网证明链路。
- 对涉及赞助、融资、收款、隐私授权的内容，在正式公开前补充主体身份、财务和合规确认。
