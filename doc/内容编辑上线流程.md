# IGNAI 内容编辑与上线流程

这一版把“后台”先压成一个稳定的 V1 内容工作流：

```text
本地内容数据 / GitHub PR / Vercel Preview / Production
```

先不要为了后台管理系统牺牲上线速度。近期活动、现场记录和加入入口能真实更新，才是 V1 最重要的闭环。

## 内容入口

当前可编辑内容集中在两个文件：

```text
src/content/events.ts
src/content/records.ts
```

封面图放在：

```text
public/images/generated
public/images/events
public/images/records
```

如果后续要把图片分类更清楚，优先新增 `public/images/events` 和 `public/images/records`，不要继续把真实活动图都放进 `generated`。

## 新增一个活动

1. 准备活动标题、状态、时间、地点、形式、简介和标签。
2. 把封面图放到 `public/images/events`。
3. 在 `src/content/events.ts` 的 `events` 数组里新增一个对象。
4. 确保 `slug` 是英文短横线格式，例如 `changsha-ai-builder-night`。
5. 本地打开 `/events` 和 `/events/[slug]` 检查排版。
6. 提交到 GitHub 后，让 Vercel Preview 先过一遍。

活动 CTA 第一版优先指向：

```text
飞书表单 / 问卷星 / Notion Form / /join
```

不要第一版就自研报名系统。

## 新增一篇现场记录

1. 准备标题、类型、时间、地点、摘要、产出标签和正文段落。
2. 把封面图放到 `public/images/records`。
3. 在 `src/content/records.ts` 的 `records` 数组里新增一个对象。
4. 本地打开 `/records` 和 `/records/[slug]` 检查移动端阅读。
5. 如果这篇记录也要发公众号 / 小红书，优先复用同一套标题、摘要和封面。

## 加入社区入口

V1 有三种方式，从轻到重：

```text
1. NEXT_PUBLIC_JOIN_FORM_URL：外部表单，最推荐
2. NEXT_PUBLIC_CONTACT_EMAIL：Email 兜底
3. Supabase：后续需要沉淀申请记录时再开启
```

如果没有配置 Supabase，`/join` 不会展示会失败的数据库表单，而是展示外部表单或 Email 入口。

## Vercel 部署

Vercel 项目建议配置：

```text
Framework Preset: Next.js
Install Command: npm install
Build Command: npm run build
Node.js Version: 22.x
```

必填环境变量：

```text
NEXT_PUBLIC_SITE_URL
NEXT_PUBLIC_CONTACT_EMAIL
```

推荐环境变量：

```text
NEXT_PUBLIC_JOIN_FORM_URL
NEXT_PUBLIC_COMMUNITY_JOIN_URL
```

V2 再配置：

```text
SANITY_PROJECT_ID
SANITY_DATASET
SUPABASE_URL
SUPABASE_SERVICE_ROLE_KEY
```

## 发布前检查

每次发布前跑：

```bash
npm run check
npm run build
```

浏览器检查：

```text
/
/events
/events/[slug]
/records
/records/[slug]
/join
/sitemap.xml
/robots.txt
```

移动端重点看：

```text
首页 Hero 和活动区
活动详情 CTA
记录详情正文
加入社区入口
```

## 何时升级到真正后台

满足任意两条，再升级 Sanity / Supabase：

```text
每周要发 3 篇以上内容
非技术成员需要自己编辑
报名人数需要结构化统计
活动照片和成员故事明显变多
需要按成员、标签、城市筛选
```

在那之前，Git 内容流 + Vercel Preview 是最稳的方案。
