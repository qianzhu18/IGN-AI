# 前端 UI 问题清单

> 创建：2026-05-29 | 状态：待修复
> 说明：当前 UI 存在较多问题，后续用其他工具集中修复。本文档记录已知问题。

---

## 已知问题

### 首页

- [ ] Hero 区域 CTA 按钮样式需打磨（扫光效果、hover 状态）
- [ ] 首页 Header 品牌锁定仍需继续微调（首屏品牌强度、移动端字重、Logo 裁切边界）
- [ ] 首页后续区块在首屏外的默认显隐过弱，容易给人“首页不完整”的感受
- [ ] 信号卡片（Signal Cards）布局在某些分辨率下可能溢出
- [ ] 成员散点区域 hover 预览卡片位置偶尔偏移
- [ ] 活动区块空状态样式需优化（目前显示"活动筹备中"占位）
- [x] 桌面端首屏品牌识别度不足：当前品牌名称、社区定位与首要行动召唤没有形成足够清晰的首屏层级
- [x] 首页首屏局部仍残留偏蓝的视觉控件，不符合 IGNAI 主色系统，需要统一回暖色品牌语言
- [x] 移动端 RIG Hero 标题行高过紧，两行中文在窄屏上挤压
- [x] 浅色模式首页活动 / 文章 section 标题和说明文字对比度过低
- [x] 活动 / 记录暗色卡片在浅色模式下被全局文字规则覆盖成深色，正文和标签不可读
- [x] 浅色模式活动 / 记录 / 文章卡片色彩基调不统一，部分页面像暗色卡片硬插在浅色背景上

### 成员系统

- [ ] 成员目录页搜索/筛选功能待实现（UI-07 后续迭代）
- [ ] 成员详情页社交链接图标在暗色模式下对比度不足
- [ ] 移动端 tap 预览卡片偶尔触发两次

### 导航

- [ ] 移动端汉堡菜单动画在低端设备上卡顿
- [ ] 菜单项 hover 状态在触摸设备上"粘滞"
- [x] 移动端菜单按钮与交互反馈存在蓝色默认态，破坏整体品牌一致性
- [x] 移动端首屏导航展开后与 Hero 内容层发生割裂，像是桌面信息直接下压到手机屏幕，缺少移动端专属的信息重排
- [x] 长页面移动端顶部导航缺少滚动方向显隐，上滑阅读和下滑找菜单的体验不够顺

### 全局

- [ ] 暗色模式下部分文字对比度不达 WCAG AA 标准
- [ ] 滚动性能在大量成员散点时有优化空间
- [ ] 字体加载闪烁（FOIT）在慢网络下明显

### /about 页面

- [ ] 动画入场时机需调整（framer-motion delay 链）
- [ ] 移动端 stats grid 布局挤压
- [x] 浅色模式价值观区标题、正文和卡片对比度过低，无法正常预览
- [x] About 页面入场 blur 在浅色模式滚动时造成文字发虚

### /events 页面

- [ ] 活动卡片信息密度在窄屏下过高

---

## 修复建议

1. 优先修复对比度和可访问性问题
2. 其次优化移动端触摸交互
3. 最后打磨动画和视觉细节
