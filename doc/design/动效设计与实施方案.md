## 一、现状诊断

### 1.1 四层评分

| 层级 | 得分 | 说明 |
|------|------|------|
| 静态视觉层 | 75/100 | 品牌色调统一，暗色主题质感好 |
| 环境动画层 | 80/100 | BackgroundFX 粒子/流线质量高，是全站最强项 |
| 结构动画层 | 55/100 | 几乎只有 Reveal 一种入场方式，缺少 Stagger、Blur In、Path Draw |
| 叙事高峰层 | 45/100 | 没有 Sticky Narrative、Background Handoff、Paper Flip 等叙事手段 |

### 1.2 核心问题

1. **动画种类集中在 Reveal + Ambient 两类** — 入场只会淡入上移，环境只有粒子循环
2. **Section 之间节奏单一** — 每个 section 都是 Reveal → 静态 → Reveal → 静态
3. **Hero 缺少文字级记忆点** — 没有 Typewriter、Blur In、Word-by-Word 等文字动画
4. **信息图式动画几乎为零** — 没有 CountUp、Marquee、SVG Path Draw、Mini Orbit
5. **交互反馈太弱** — 卡片 hover 只有 -1px 上浮，没有 Spotlight、Magnetic、Shimmer

### 1.3 已有优势（保持）

- BackgroundFX.tsx — Canvas 粒子 + 流线 + 脉冲，质量高于 90% 的社区网站
- Reveal.tsx — 带 brightness/contrast 滤镜的淡入，比普通 opacity+y 有质感
- Hero 视差滚动 — scrollYProgress → useTransform
- CSS hover 上浮 + 边框发光 — energy-card 组件
- converge-ray CSS 光线汇聚 — Join CTA 装饰

## 二、品牌动效语言

> 全站动效应围绕四个关键词统一：**Glow（发光）/ Drift（漂移）/ Reveal（揭示）/ Pulse（脉冲）**

| 关键词 | 情绪 | 技法 | 使用位置 |
|--------|------|------|----------|
| **Glow** | 能量、点燃 | 边框发光、光斑跟随、shimmer 扫光 | CTA 按钮、卡片 hover、Hero 能量核 |
| **Drift** | 流动、信号传递 | 浮动球体、流线、Marquee 滚动 | 背景、信息流、合作伙伴展示 |
| **Reveal** | 层次、叙事 | 淡入上移、模糊渐显、交错出场 | 内容入场、section 标题 |
| **Pulse** | 活力、心跳 | 呼吸脉冲、CountUp、实时指示灯 | Live 点、数据统计、状态指示 |

## 三、各 Section 动效方案

### 3.1 Hero

**现状**: AOS zoom-y-out 入场 + 渐变文字 + 静态 SVG 背景
**问题**: 标题没有逐字/逐行动画，SVG 背景全静态，CTA 按钮无光效

**改进方案**:

| 层 | 动效 | 技法 | 参数 |
|----|------|------|------|
| 背景层 | 能量核心发光球 | Canvas 或 CSS blur orbs | 3-4 个球，14-20s 循环 |
| 文字层 | 标题逐字 Blur In | filter: blur(8px)→0, 800ms | 配合 stagger 100ms |
| CTA 层 | 按钮微光扫过 | Shimmer 光效 | 4s 循环，rgba(255,255,255,0.22) |
| 滚出 | 缩小+淡出 | scale 1→0.94, opacity 1→0.75 | scroll 驱动 |

### 3.2 What is IGNAI

**现状**: 左右双栏 + Reveal 淡入
**问题**: 入场太统一，缺乏视觉焦点

**改进方案**:

| 层 | 动效 | 技法 | 参数 |
|----|------|------|------|
| 背景 | 轨道关系图 | SVG Orbit 旋转 | 3-5 个节点，12s 循环 |
| 内容 | 关键词标签交错入场 | Stagger + Fade | delay: i * 0.1 |
| 卡片 | 成员组成卡 hover | 上浮 + 边框发光 | translateY(-4px) + shadow |

### 3.3 Why Now

**现状**: Reveal + 静态卡片
**问题**: 中间 section 缺少「结构高潮」

**改进方案 — Sticky Narrative**:

| 层 | 动效 | 技法 | 参数 |
|----|------|------|------|
| 左侧 | 固定标题随滚动切换 | CSS sticky + state 驱动 | position: sticky, top: 40% |
| 右侧 | 卡片逐步 takeover | opacity + scale + blur | active: scale 1, blur 0; inactive: scale 0.96, blur 4px |
| 色彩 | 每张卡对应一种情绪色 | border-color 渐变 | 好奇=橙, 表达=紫, 行动=金, 协作=蓝 |

### 3.4 Culture

**现状**: Reveal + 卡片 hover 上浮 -1px
**问题**: stagger 间隔太短(80ms)，卡片 hover 太弱

**改进方案**:

| 层 | 动效 | 技法 | 参数 |
|----|------|------|------|
| 入场 | 加大 stagger 间隔 | delay: i * 0.15-0.2 | 节奏更舒缓 |
| hover | 增强反馈 | translateY(-4px) + border brighten + shadow deepen | 比 -1px 明显 |
| 背景 | 深暗渐变 + 淡色光雾 | radial-gradient + low opacity | 氛围层 |

### 3.5 Upcoming Events / Community Roles

**现状**: Reveal + 极弱 hover
**问题**: 卡片间无差异，缺少信息流动态

**改进方案**:

| 层 | 动效 | 技法 | 参数 |
|----|------|------|------|
| hover | Spotlight 跟随鼠标 | radial-gradient 随鼠标位置 | rgba(255,255,255,0.08), 60% radius |
| 时间 | 活动卡日期高亮动画 | border-glow pulse | 2s 循环 |
| 角色卡 | 悬浮展开详情 | height/opacity 展开动画 | 300ms, ease-out |

### 3.6 Join CTA

**现状**: converge-ray CSS 光线 + 普通 CTA
**问题**: 光线只在底部可见，CTA 缺少吸引力

**改进方案**:

| 层 | 动效 | 技法 | 参数 |
|----|------|------|------|
| 背景 | 流动能量线汇聚 CTA | SVG Path Draw 或 Canvas | 向按钮中心汇聚 |
| 按钮 | Magnetic 效果 | 按钮 hover 向鼠标偏移 4-12px | 仅 CTA 按钮 |
| 标题 | 双行渐进揭示 | 第一行先出，第二行延迟 0.3s | 配合背景亮度脉冲 |
| 社交入口 | 单个高亮 + 图标放大 | scale 1.04 + border glow | 500ms |

### 3.7 Footer

**现状**: 纯静态，无任何动画
**问题**: 收尾太草率

**改进方案**:

| 层 | 动效 | 技法 | 参数 |
|----|------|------|------|
| 分隔线 | 流动渐变光条 | linear-gradient + animation | 3s 循环 |
| 链接 | 下划线展开 | width: 0→100%, hover 触发 | 300ms |
| 底部 | 版权文字渐变流动 | background-position 动画 | 极慢循环 |

## 四、动画技法速查（本站适用）

### 高优先级技法

| 技法 | 类别 | 用在哪 | 实现 |
|------|------|--------|------|
| Blur In | 入场 | Hero 标题 | Framer Motion: filter: blur(8px)→0 |
| Stagger | 入场 | 所有卡片列表 | delay: i * 0.15 |
| Shimmer | 循环 | CTA 按钮 | motion.span, x: [-120%, 140%] |
| Spotlight | 交互 | 卡片 hover | radial-gradient 跟随鼠标 |
| Sticky Narrative | 叙事 | Why Now | CSS sticky + state 驱动 |
| Marquee | 叙事 | 合作伙伴/标签 | 复制内容 + x: [0%, -50%] |
| SVG Path Draw | 结构 | 流程图/能量线 | pathLength: 0→1 |
| Magnetic Button | 交互 | Join CTA | 鼠标偏移 4-12px |

### 中优先级技法

| 技法 | 类别 | 用在哪 |
|------|------|--------|
| Floating Orbs | 循环 | 背景氛围球 |
| CountUp | 数据 | 统计数字 |
| Typewriter | 文本 | Hero 关键词轮播 |
| Brand Reveal | 文本 | AI 回答品牌高亮 |
| 3D Flip | 叙事 | 范式对比 |
| Layout Animation | 交互 | 导航标签 |

## 五、技术选型

### 当前技术栈

| 技术 | 状态 | 用途 |
|------|------|------|
| AOS (CDN) | 已安装 | 滚动触发动画（landing 主题用） |
| Tailwind CSS | 已安装 | CSS transition / hover |
| @headlessui/react | 已安装 | Transition 组件 |
| Canvas API | 已用 | BackgroundFX 粒子 |

### 需要引入

| 技术 | 用途 | 方式 |
|------|------|------|
| **framer-motion** | 90% 的动画需求 | `yarn add framer-motion` |
| Tailwind CSS | 补充 CSS 动画 | 已有，无需安装 |
| CSS Keyframes | 循环动画 | 在 style.js 中定义 |

### 不需要引入（当前阶段）

- GSAP — framer-motion 覆盖 90% 需求，暂不需要
- Lenis — 平滑滚动是锦上添花，不影响动画核心
- React Three Fiber — 3D 能量核心属于 P3 优先级
