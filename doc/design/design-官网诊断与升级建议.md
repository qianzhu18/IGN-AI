# IGNAI 官网诊断与升级建议

> 基于代码审查 + 页面实际渲染 + 行业对标的三维诊断
> 诊断日期：2026-05-06
> 诊断范围：`http://localhost:3003/` 首页全链路

---

## 一、总体评价

**当前状态：7 分 / 10 分** — 框架扎实，视觉基调到位，但缺少"让人记住"的签名时刻。

具体来说：
- 技术架构和代码组织是**专业级**的，分层清晰
- 视觉基调（深夜黑蓝 + 火种橙金 + Canvas 粒子）已有**高级感**
- Canvas 背景特效（粒子 + 流线 + 脉冲 + 收敛）是**亮点**
- 但整体体验偏向"安静优雅"，缺少"哇"的瞬间
- 首屏冲击力不足，滚动叙事平淡，交互反馈偏弱

用一句话形容当前感觉：**"一个气质很好的页面，但还没有开始表演。"**

## 二、逐层诊断

### 2.1 首屏 Hero（最关键，最需提升）

**现状**：
- 左侧：eyebrow 标签 + "IGNAI" + 大标题 "Ignite before AGI." + 副标题 + CTA
- 右侧：AI 生成的城市/光线图 + 3 个信号指示器
- 有 Framer Motion 视差（contentY / imageY / imageScale）
- 有 ignite-field 能量场呼吸动画

**问题诊断**：

| 问题 | 严重度 | 说明 |
|------|--------|------|
| 标题冲击力不够 | 高 | "Ignite before AGI." 用 Cormorant Garamond 衬线体显示，字号够大但"点燃"感没出来。没有配合文字动画（逐词入场、模糊渐清、渐变流动），就是一个静态大字 |
| Hero 图片缺乏质感 | 高 | AI 生成的 `local-global-embers.png` 像一张普通插图，不是"签名视觉"。缺少动态感（只有 slow-pan 微移），没有粒子覆盖、没有光线扫过 |
| CTA 区域不够聚焦 | 中 | 两个按钮并排，视觉权重分散。"加入社区"主按钮有 brand gradient，但"查看近期活动"作为次要按钮不够有吸引力 |
| 3 个信号指示器存在感弱 | 中 | 底部 Local roots / Global signal / Human warmth 三栏信息太"安静"，像补充说明而非核心亮点 |
| 导航栏过于简单 | 中 | 只有 logo + 5 个文字链接，无 hamburger 移动端菜单，无滚动 sticky 行为，无当前页高亮 |

**升级建议**：

1. **标题动画化**：至少做以下之一
   - 逐词淡入 + 上浮（每个词延迟 80ms）
   - 模糊渐清（`filter: blur(8px)` → 清晰，800ms）
   - 渐变文字流动（`background-clip: text` + `background-position` 动画）
   - 理想方案：标题 + 粒子爆发组合（标题出现时，粒子从标题位置向外扩散）

2. **Hero 视觉层升级**：
   - 给 Hero 图片叠加一层 Canvas 效果（光点沿图片中的光线移动）
   - 或替换为有 3D 深度的视觉效果（Three.js 能量核 / 粒子聚合形态）
   - 至少给图片加一层动态渐变遮罩（颜色随时间微微变化）

3. **CTA 按钮升级**：
   - 主 CTA 持续微脉冲（已有 `cta-heat`，确认生效）
   - 添加磁性吸附效果（鼠标接近时按钮微微被吸引）
   - 按钮出现时有弹性入场（`scale: 0 → 1.05 → 0.98 → 1`）

4. **导航栏升级**：
   - 添加滚动 sticky 行为（滚出 Hero 后，导航固定在顶部，背景变毛玻璃）
   - 添加移动端 hamburger 菜单
   - 添加当前 section 高亮（滚动时导航项对应高亮）

### 2.2 What is IGNAI Section

**现状**：
- 左侧 sticky 文案 + 右侧 4 张成员卡片 open-grid
- Reveal 入场动画
- 卡片有 presence-card 呼吸效果

**问题**：
- 文案和卡片是"平铺直叙"，缺少叙事编排
- sticky 文案只是固定在那里，不会随滚动变化内容
- 4 张卡片同时出现，没有利用滚动逐步揭示的能力

**升级建议**：
- 改为 **Sticky Narrative 模式**：左侧 sticky 区域随滚动进度变化内容（先显示标题 → 再显示副标题 → 再显示描述），右侧卡片随滚动逐个出现
- 每张卡片出现时加 **Scale In + Blur In** 组合，比单纯的 Reveal Up 更有层次

---

### 2.3 Culture Section（"不是围观，是上场"）

**现状**：
- 4 张文化卡片 stagger 入场（index × 0.08s 间隔）
- presence-card 呼吸效果
- 卡片内容：先行动 / 先表达 / 先上场 / 先点火

**问题**：
- 4 张卡片的 stagger delay 只有 0.08s（80ms），视觉上几乎是同时出现，stagger 效果不明显
- 卡片是纯文字 + 呼吸，没有图形或图标元素，视觉单调
- 没有图标/emoji/编号以外的视觉锚点

**升级建议**：
- Stagger delay 增大到 **150-200ms**，让交错感更明显
- 每张卡片增加一个**动态图标或微型动画**（如"先行动"配一个脉冲箭头、"先点火"配一个闪烁火花）
- 卡片 hover 时增加 **3D Tilt 效果**（D2 类），鼠标跟随倾斜 + 光斑

---

### 2.4 Upcoming Events Section

**现状**：
- 3 列卡片网格，EventCard 组件
- 图片 hover 时 scale(1.03)
- 卡片 hover 上浮 -1px + 边框变亮

**问题**：
- 3 张卡片使用同一张 AI 生成的占位图（不同文件但风格类似），缺乏真实感
- 卡片的 hover 效果过于克制（-1px 几乎感知不到）
- 没有"即将开始"的紧迫感视觉提示

**升级建议**：
- 卡片 hover 上浮增大到 **-4px 到 -6px**
- 增加**倒计时/倒计时徽章**（如果是未来活动）
- 图片 hover 时除了 scale，再加一个亮度/对比度变化
- 如果没有真实图片，用**渐变色块 + 活动类型图标**替代 AI 占位图，反而更专业

---

### 2.5 Field Notes Section

**现状**：
- 与 Events 类似的卡片网格布局
- 首条 featured 占两列

**问题**：
- 与 Events Section 视觉上太相似，没有差异化
- Records 和 Events 用几乎相同的卡片样式，用户可能混淆

**升级建议**：
- Field Notes 卡片用**不同的视觉语言**（例如更偏文档/笔记风格，不像活动卡片那么"宣传"）
- Featured 记录可以加一个**时间线连接线**效果，暗示"这是沉淀下来的"
- 考虑用**不同的 hover 效果色**（Records 用蓝色系，Events 用橙色系）

---

### 2.6 Community Roles Section

**现状**：
- 与 What is IGNAI 几乎相同的 open-grid 布局
- 4 张角色卡片 + Reveal stagger

**问题**：
- 与 What is IGNAI 的"成员分类"视觉上重复（都是 open-grid 4 卡片）
- 信息密度低，每张卡片只有标题 + 一句话描述

**升级建议**：
- 考虑改用**不同布局**：圆形布局（4 个角色围绕中心 logo）或水平滚动展示
- 增加角色**图标或头像占位**
- 鼠标 hover 时展示更多信息的展开效果

---

### 2.7 Join Section（CTA 收尾）

**现状**：
- converge-ray 收敛流线动画（3 条线汇聚）
- CTA 按钮有 cta-heat 呼吸脉冲
- 社交链接 slots

**问题**：
- converge-ray 效果只有在页面底部才能看到（需要滚动到 68%+），而此时用户已经被前面的内容吸引走了
- 社交链接如果是灰显占位，显得不完整

**升级建议**：
- Join Section 的背景氛围应该**明显区别于其他 section**（用背景接管 E7，让这里变成一个"情绪高潮点"）
- CTA 按钮考虑更大、更突出（可以比首屏 CTA 更大，因为是最后转化机会）
- 如果社交链接还没配置，用"即将开放"的微动效标签替代灰显

---

### 2.8 Footer

**现状**：简洁的双列布局，品牌名 + slogan + 导航 + 联系方式

**问题**：没有动效，和其他 section 的精致度不匹配

**升级建议**：
- Footer 上方加一条渐变分割线（带流光效果）
- Footer 导航项 hover 加微交互（下划线展开 / 颜色渐变）
- 底部 copyright 可以加一个微弱的渐变文字流动

## 三、全局性问题

### 3.1 缺少全局 SiteHeader ⚠️ 高优先级

**现状**：导航栏嵌在 HeroSection 内部，滚动后导航消失。子页面（/events, /records, /join 等）完全没有顶部导航。

**影响**：用户在子页面时无法回到首页或跳转到其他页面。

**建议**：
- 创建独立的 `SiteHeader` 组件
- Hero 区域的导航保留作为"首屏导航"
- 滚动超过 Hero 后，顶部出现一个 sticky 的全局导航（毛玻璃背景 + logo + 简化导航项）
- 移动端添加 hamburger 菜单

### 3.2 缺少页面转场动画

**现状**：页面切换时没有过渡，瞬间跳转。

**建议**：
- 使用 Framer Motion 的 `AnimatePresence` + `motion.main` 实现淡入淡出
- 或使用 Next.js App Router 的 `template.tsx` 实现页面级动画
- 至少做一个 opacity fade（300ms），高级版做 shared layout transition

### 3.3 缺少平滑滚动

**现状**：使用原生 `scroll-behavior: smooth`，只有锚点跳转时平滑，普通滚动不丝滑。

**建议**：引入 Lenis 实现全局平滑滚动。这是提升"高级感"成本最低的方式之一。

### 3.4 Design Token 系统不完整

**现状**：
- `tokens.css` 定义了基础变量但没覆盖所有使用场景
- Tailwind 配置中没有映射品牌色
- 大量硬编码色值散落在组件中（`#ff9a3c`, `#0d1118`, `#f0c78e` 等出现多次）

**建议**：
- 将所有品牌色映射到 Tailwind 配置
- 建立完整的 `colors / spacing / typography / shadows / animation` token 体系
- 组件中使用 `text-accent-orange` 而非 `text-[#ff9a3c]`

### 3.5 首页缺少 4 个已设计的 Section

**现状**：`WhyNowSection`, `IdentitySection`, `TraitsSection`, `ShowcaseSection` 已有代码文件但首页未引入。

**影响**：品牌叙事不完整。缺少"Why now"品牌宣言（设计文档中最重要的叙事段）和"IGNAI / 洋来社"双身份展示。

**建议**：分批上屏，优先 `WhyNowSection`（品牌宣言 → 说服力）和 `IdentitySection`（双身份 → 认知度）。

### 3.6 移动端体验不完整

**现状**：
- 导航只在 `lg:` 以上显示
- 没有 hamburger 菜单
- 部分 section 在小屏上信息层级不够清晰

**建议**：
- 添加移动端导航方案（hamburger / 底部 tab bar）
- 重新审视移动端的 section 排列顺序和内容优先级

### 3.7 OG 图片格式问题

**现状**：`/og/ignai-og.svg` 是 SVG 格式，但 Open Graph 协议推荐 PNG/JPG，部分社交媒体平台不支持 SVG OG 图片。

**建议**：转换为 PNG 格式（1200×630 标准）。

## 七、最终一句话

> IGNAI 官网现在的状态是**"骨架优美，但肌肉还没长出来"**。技术架构和视觉基调是好的，缺的是：签名时刻（Hero 3D 能量核）、叙事编排（Sticky Narrative）、和交互精致度（光斑追踪 / 鼠标视差 / 磁性按钮）。Phase 1 的 6 个改动可以在 2 天内完成，效果立竿见影。
